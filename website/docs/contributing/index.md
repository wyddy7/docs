# Руководство для авторов (Contributing)

Добро пожаловать! Мы рады, что вы хотите внести вклад в проект учебных материалов.

Это руководство поможет вам понять, как правильно вносить изменения в документацию проекта.

## С чего начать

Если вы впервые работаете с проектом, начните с раздела [Как внести вклад](./getting-started.md). Там вы найдете пошаговую инструкцию по созданию Pull Request.

## Содержание документации

- **[Как внести вклад](./getting-started.md)** — пошаговая инструкция для студентов по созданию Pull Request из форка
- **[Стандарты оформления документации](./documentation-standards.md)** — правила именования файлов, обязательные секции и шаблоны для лабораторных и курсовых работ
- **[Работа с файлами](./working-with-files.md)** — работа со страницей "Полезные ссылки", добавление ZIP архивов и работа с листингами кода
- **[Комментарии и обсуждения](./comments.md)** — использование системы комментариев через GitHub Discussions
- **[Руководство администратора](./admin-guide.md)** — простое руководство для преподавателя по работе с системой

## Быстрая навигация

- Нужно исправить опечатку? → [Как внести вклад](./getting-started.md)
- Создаете новую лабораторную работу? → [Стандарты оформления документации](./documentation-standards.md)
- Нужно добавить файлы для скачивания? → [Работа с файлами](./working-with-files.md)
- Есть вопрос по документации? → [Комментарии и обсуждения](./comments.md)
- Преподаватель и нужно отредактировать материалы? → [Руководство администратора](./admin-guide.md)
