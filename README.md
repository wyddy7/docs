# Документация лабораторных работ

Сборник методических материалов для выполнения лабораторных работ и курсовых проектов по дисциплине "Организация ЭВМ и систем", "Периферийные устройства".

## Для студентов

Все методички собраны в одном месте с пошаговыми инструкциями, примерами кода (в некоторых работах) и **самым важным — листингами и приложениями**, которые содержат готовые решения и программы на Assembly и VHDL.

## Для преподавателей

Если вы хотите редактировать материалы, ознакомьтесь с [руководством администратора](website/docs/contributing/admin-guide) — простым руководством по работе с системой.

## Для контрибьюторов

Если вы хотите помочь улучшить материалы или добавить новую информацию (студенты, аспиранты, другие преподаватели), изучите [руководство по контрибьюции](website/docs/contributing/). Там описаны:
- Процесс создания Pull Request
- Стандарты оформления документации
- Работа с ZIP архивами и компонентами
- Шаблоны для лабораторных и курсовых работ

## Комментарии и обсуждения

На страницах лабораторных работ, курсовых проектов, а также на страницах "Ссылки" и "Контрибуция" доступна система комментариев через GitHub Discussions. Вы можете:

- **Задавать вопросы** по лабораторным работам и получать помощь от других студентов
- **Делиться решениями** и полезными советами
- **Сообщать об ошибках** в методичках
- **Обсуждать** сложные моменты выполнения работ

**Важно:** Пожалуйста, соблюдайте [правила сообщества](website/docs/community-guidelines.md) при участии в обсуждениях. Комментарии модерируются, и нарушение правил может привести к блокировке.

## Быстрый старт

```bash

```



Собранный сайт будет в папке `website/build/`.

## Деплой на GitHub Pages

### Автоматический деплой через GitHub Actions

При пуше в ветку `main` автоматически:
- Собирается сайт (используются замороженные версии зависимостей для стабильности)
- Деплоится в ветку `main` (GitHub Pages настроен на эту ветку)

**Важно:** Версии Node.js и GitHub Actions заморожены в workflow файле для обеспечения стабильности сборки в будущем.

### Настройка GitHub Pages

1. Зайди в **Settings → Pages** репозитория
2. Выбери:
    - **Source**: Deploy from a branch
    - **Branch**: `main`
    - **Folder**: `/website/build`
3. Сохрани

Сайт будет доступен по адресу: `https://<username>.github.io/docs/`

### Ручной деплой

```bash
cd website
npm run build
# Затем закоммитьте и запушьте папку website/build в ветку main
```

## Настройка

Перед деплоем обнови в `website/docusaurus.config.ts`:

-   `url` - URL вашего сайта
-   `organizationName` - ваш GitHub username
-   `projectName` - название репозитория

Подробнее о настройке компонентов см. [website/README.md](website/README.md)

## Полезные команды

```bash
# Разработка (из папки website/)
cd website
npm start              # Запуск dev сервера (http://localhost:3000)
npm run build          # Сборка для production
npm run serve          # Просмотр собранного сайта локально

# Утилиты
npm run clear          # Очистка кэша
npm run typecheck      # Проверка типов TypeScript
npm run create-zip     # Создание ZIP архива (из папки website/)
```

### Создание ZIP архивов

Для создания ZIP архивов из папок используется скрипт `scripts/create-zip.js`:

```bash
# Из корня проекта
node scripts/create-zip.js <исходная_папка> <выходной_файл>

# Или через npm скрипт (из папки website/)
cd website
npm run create-zip
```
